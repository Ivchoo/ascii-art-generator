# 🎨 ASCII Art Generator

> Курсова работа по **Операционни системи**

Трислойно (микроуслужно) уеб приложение, което приема въведен от потребителя текст и го трансформира в голям ASCII арт символ. Проектът е напълно контейнеризиран чрез **Docker** и се управлява централизирано с **Docker Compose**.

---

## 📁 Структура на проекта

```
os-ascii-project/
├── backend/
│   ├── app.py                  # Главен код на Flask API-то
│   ├── Dockerfile              # Инструкции за контейнеризация на бекенда
│   └── requirements.txt        # Python зависимости (Flask, Redis, pyfiglet, etc.)

│   ├── index.html              # Потребителски интерфейс и JavaScript логика
│   ├── nginx.conf              # Настройки на Nginx и Reverse Proxy конфигурация
│   └── Dockerfile              # Инструкции за контейнеризация на фронтенда
└── compose.yml                 # Главен Docker Compose файл за стартиране на мрежата
```

## ⚙️ Компоненти на приложението

Приложението се състои от **три услуги**, работещи в пълен синхрон:

### 🌐 Frontend — Nginx (Уеб Сървър)

- Използва лек и бърз HTTP сървър **Nginx**
- Сервира статичния потребителски интерфейс (`index.html`) на **порт 8080**
- Действа като **Reverse Proxy** — приема клиентските заявки от браузъра и ги пренасочва към бекенда, скривайки вътрешната инфраструктура от крайния потребител

### 🐍 Backend — Flask/Python (Бизнес Логика)

- Изграден на **Python** с микро-рамката **Flask**, работи на **порт 5000**
- Приема текстовия вход и използва библиотеката **pyfiglet** за трансформация на буквите в ASCII арт
- Преди всяка генерация проверява за кеширан резултат, за да избегне излишен процесорен ресурс

### 🗄️ Cache — Redis (Временна памет)

- **In-memory** база данни от тип "ключ-стойност", работеща на **порт 6379**
- Запазва генерираните ASCII резултати за **60 секунди**
- Осигурява мигновено връщане на резултата (< 2ms) при повторно търсене на същата дума

## 🔄 Комуникация между услугите

Всички услуги комуникират вътре в **изолирана виртуална мрежа**, автоматично създадена от Docker Compose, чрез вградената DNS система на Docker.

```
Потребител
    │
    │  http://localhost:8080
    ▼
┌─────────────┐
│   Frontend  │  (Nginx — порт 8080)
│   Nginx     │  Сервира UI, обратно проксира /api/generate
└──────┬──────┘
       │  http://backend:5000/generate
       ▼
┌─────────────┐
│   Backend   │  (Flask — порт 5000)
│   Flask     │  Генерира ASCII арт с pyfiglet
└──────┬──────┘
       │  redis-cache:6379
       ▼
┌─────────────┐
│    Cache    │  (Redis — порт 6379)
│    Redis    │  Кешира резултати за 60 секунди
└─────────────┘
```

### Поток на заявката

1. **Потребител → Frontend:** Браузърът зарежда `http://localhost:8080`. При въвеждане на текст JavaScript изпраща заявка към `/api/generate?text=...`
2. **Frontend → Backend:** Nginx прихваща `/api/generate` и пренасочва трафика към `http://backend:5000/generate`
3. **Backend → Cache:** Flask проверява Redis за кеширан резултат
   - ✅ **Намерен в кеша** → връща се мигновено с `cached: true`
   - ❌ **Не е намерен** → Flask генерира арта, записва го в Redis и връща резултата с `cached: false`

> Бекендът връща **JSON отговор** с ASCII съдържанието и булева променлива `cached`, за да може интерфейсът да информира потребителя за източника на данните.

## 🚀 Стартиране на проекта

### Системни изисквания

- **Docker Desktop** (Windows / macOS) или **Docker Engine + Docker Compose** (Linux / WSL)

### Стъпки

**1. Навигирайте до главната папка на проекта:**

```bash
cd os-ascii-project
```

**2. Изградете образите и стартирайте контейнерите:**

```bash
docker compose up --build
```

> ⚠️ Флагът `--build` е задължителен при първоначално стартиране или след направени промени по сорс кода.

**3. Отворете приложението в браузъра:**

```
http://localhost:8080
```

### Спиране на приложението

За да спрете контейнерите и да почистите мрежата:

```bash
# Натиснете Ctrl + C в текущия терминал, или изпълнете в нов прозорец:
docker compose down
```
